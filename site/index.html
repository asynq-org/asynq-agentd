Open-source daemon runtime

asynq-agentd

Local daemon and CLI for running and observing coding agents on your own machine. Built to power Asynq Buddy, but useful on its own today.

View on GitHub Read the docs

What it already does

  • Runs Claude Code and Codex through a local daemon
  • Queues tasks, approvals, recurring jobs, and recent-work continuation
  • Exposes REST, SSE, WebSocket, and terminal-oriented control surfaces
  • Ships with a local operator CLI and bootstrap installers

CLI-first today

The open-source repo focuses on the daemon and operator CLI. The full Buddy web and mobile UI live in a separate product repo.

Local-first architecture

Pair a machine you control, keep runtime state local, and use Tailscale or your own network path to reach it remotely.

Beta by design

The runtime is already useful, but some adapters, packaging, and onboarding flows are still being hardened in public.

Install

curl -fsSL https://agentd.asynq.org/install.sh | sh
irm https://agentd.asynq.org/install.ps1 | iex

Today the hosted entrypoints are thin wrappers: they clone the repo into a temporary directory and run the real bootstrap installer from source.

Open-source scope

`asynq-agentd` is the daemon. Asynq Buddy is the user-facing product built on top of it. That split lets the runtime stay open while the higher-level app evolves separately.
